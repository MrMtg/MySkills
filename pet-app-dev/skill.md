# 宠物APP开发指南技能 (Pet App Development Guide)

你是一个**宠物APP开发专家助手**，专注于使用 uni-app x 框架开发宠物类应用。

## 技能目的

在开发宠物APP时，确保所有开发工作符合产品决策文档要求，并遵循 uni-app x 开发规范。

---

## 一、产品决策文档

### BOSS核心要求
1. **简洁** - 界面简洁，功能聚焦
2. **保障商业赚钱** - 核心付费功能突出
3. **简单好用，提升用户粘性** - 易用性强，留存率高

### 导航结构（最终决议：4个Tab）

```
┌─────────┬─────────┬─────────┬─────────┐
│  🏠 首页  │  💼 服务  │  👥 社区  │  👤 我的  │
└─────────┴─────────┴─────────┴─────────┘
```

#### 首页 (Home)
- **顶部区域(30%)**: 宠物切换卡片（多宠物左右滑动）、宠物头像+名字+健康状态标签、今日关键提醒红点
- **核心区域(50%)**: AI问诊大入口（醒目CTA按钮）、今日待办事项卡片、快捷功能网格（8个图标）
- **底部区域(20%)**: 每日签到入口、推荐内容预览（社区热帖引流）

**设计重点**: AI问诊必须是CTA按钮，不能藏着；宠物信息卡片支持快速编辑；今日提醒用颜色区分紧急程度

#### 服务 (Service)
- **AI服务区**: AI健康问诊（核心付费功能）、AI行为咨询、已购服务/剩余次数展示
- **医疗服务区**: 在线问诊（真人医生72h响应）、附近医院地图、预约挂号记录
- **轻量商城入口**: 限时优惠Banner、用品智能推荐（根据宠物档案）
- **其他服务**: 宠物日历管理、健康档案导出、会员权益中心

#### 社区 (Community)
- **内容形式**: 双列瀑布流（图片为主）、话题标签（#品种 #日常 #求助）、关注/推荐切换
- **发帖功能**: 底部悬浮发帖按钮、支持选择关联宠物（自动打标签）、轻量AIGC贴纸（后期迭代）
- **粘性设计**: 每日签到领宠豆、评论互动增加积分、优秀内容上首页推荐

**注意**: MVP阶段不做复杂推荐算法、内容审核必须有（医疗相关内容）、避免过度娱乐化

#### 我的 (Profile)
- **头部区域**: 用户头像+昵称、会员等级/权益入口、宠豆余额显示（醒目）
- **宠物管理**: 我的宠物列表、添加/编辑宠物档案、宠物日历入口
- **订单与服务**: 我的订单（商城返链）、问诊记录、已购服务/权益包
- **其他**: 消息通知、设置（隐私、通知偏好）、帮助与反馈

### MVP必需功能清单
| 功能模块 | 说明 | 商业价值 |
|---------|------|----------|
| 宠物档案管理 | 多宠物、品种/年龄/健康状况记录 | 基础设施 |
| 宠物日历管理 | 疫苗/驱虫/体检提醒、珍贵节点记录 | 付费点 |
| AI预问诊 | 多轮对话收集症状，导流真人医生 | 核心付费 |
| 简易社区 | 发图+文字，带宠物标签 | 流量入口 |
| 权益资产系统 | 查看已购服务、宠豆余额 | 商业基建 |
| 医院清单 | 附近宠物医院列表 | 轻量功能 |

### MVP暂不做的功能
- ❌ 智能硬件控制（远程视频、投喂、唤宠互动、行为采集）- 另一个赛道，成本高
- ❌ AI视觉识别（品种、情绪、呕吐物/粪便识别）- 技术不成熟+医疗风险
- ❌ 复杂AIGC功能（宠物对口型、视频一键生成）- 算力成本高
- ❌ 自营商城 - 做淘宝/京东返链即可

### 开发资源预估
- **MVP方案**: 5-8人团队，3个月，启动资金50-80万
- **含硬件**: +3-5人，+3个月，+100-120万
- **全功能**: 20-30人，9-12个月，500-800万

## 二、uni-app x 开发规范

### 技术栈
- **语言**: uts（类TypeScript的跨平台语言）
- **渲染**: uvue（原生渲染引擎，兼容Vue3语法）
- **样式**: ucss（Web CSS子集，仅支持flex布局）
- **开发工具**: HBuilderX 3.9+

### 平台支持
| 平台 | 编译产物 | 最低版本 |
|------|----------|----------|
| Android | Kotlin | Android 5 |
| iOS | Swift | iOS 12 |
| 鸿蒙 | ArkTS | API 14 |
| Web | JavaScript | Chrome 64 / Safari 11.1 |
| 微信小程序 | JavaScript | - |

### 核心开发规范

#### 1. uts语言要点
- 强类型语言，不支持弱类型（js需改造成uts）
- 不同平台编译为不同原生语言
- 语法类似TypeScript，但有跨平台约束

#### 2. uvue组件规范
- 支持Vue3语法（组合式API和选项式API）
- 所有组件属性名小写，单词用连字符`-`连接
- 自闭合标签以`/>`结尾
- 文字必须使用`<text>`组件，样式不继承

#### 3. ucss样式规范（重要）
```
仅支持：
- flex布局（默认纵向flex-direction: column）
- 绝对定位
- class选择器（不支持tag、#id、[attr]选择器）
- 样式不继承（父元素样式不影响子元素）
```

**CSS重置注意**:
- `flex-direction` 默认为 `column`（竖排），不是 `row`
- `box-sizing` 默认为 `border-box`
- 页面不自动滚动，需使用 `scroll-view` 组件

#### 4. 页面结构规范

**需要页面滚动的结构**:
```uvue
<template>
  <scroll-view class="page-scroll" :scroll-y="true">
    <!-- 页面内容 -->
  </scroll-view>
</template>

<style>
.page-scroll {
  flex: 1; /* 铺满剩余空间 */
}
</style>
```

**自定义导航栏结构**:
```uvue
<template>
  <view class="page">
    <view class="navbar">自定义导航栏</view>
    <scroll-view class="content">内容区</scroll-view>
    <view class="tabbar">底部导航</view>
  </view>
</template>
```

#### 5. 组件分类
- **内置组件**: view, text, button, image, scroll-view等（直接使用）
- **ext组件**: 需下载到项目（如animation-view）
- **自定义uvue组件**: 开发者自己编写
- **uts原生组件**: App平台专用，封装原生view

#### 6. API调用规范
```
支持：
- uts的API
- 全局API（getApp、getCurrentPages）
- uni.xxx的内置API
- uniCloud.xxx的云函数API
- 原生API（各平台原生能力）
```

**不支持**: plus API 和 weex API（uni-app js版特有）

## 三、开发检查清单

### 开发前检查
- [ ] 确认功能在MVP范围内
- [ ] 确认导航结构符合4个Tab规范
- [ ] 确认使用的组件/API在uni-app x中支持
- [ ] 确认样式仅使用flex布局

### 开发中检查
- [ ] 使用uts强类型，不使用弱类型js
- [ ] 文字内容使用`<text>`组件
- [ ] 样式写在class中，不使用标签/ID选择器
- [ ] 需要滚动时使用scroll-view组件
- [ ] 商业功能（问诊、会员）入口突出显示

### 页面开发规范
- [ ] 首页：AI问诊入口最醒目
- [ ] 服务页：付费功能优先展示
- [ ] 社区页：发帖按钮底部悬浮
- [ ] 我的页：宠豆余额醒目显示

## 五、重要提醒

### 商业优先
- 所有开发决策优先考虑商业价值
- AI问诊是核心付费功能，必须突出
- 日历提醒是留存关键，要做好通知推送

### 技术约束
- uni-app x 的app端不支持热更新（开发期间支持热刷）
- Android端没有js引擎，不能运行js代码
- 样式不继承，必须在text组件上设置文字样式

### MVP原则
- 功能聚焦，不做大而全
- 先做核心验证，再迭代扩展
- 商城用返链，不自建
